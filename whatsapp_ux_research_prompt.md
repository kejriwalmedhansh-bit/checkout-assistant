I'm designing the WhatsApp conversation flow for "Dealo," a pre-checkout purchase-optimization tool for Indian consumers. A user sends a product name or a product link on WhatsApp. The bot searches multiple retailers, checks for stackable Gyftr gift-voucher discounts, and checks for cashback credit card bonuses, then replies with the single cheapest legitimate way to actually buy that product — not a coupon list, a specific purchase route.

CURRENT MESSAGE STRUCTURE (what I'm trying to improve):
A typical successful reply today is 3 separate WhatsApp message bubbles, sent back-to-back with no user action in between:
- Bubble 1 (text): product title, a "RECOMMENDED ROUTE" label, the merchant name, the final price crossed out against the original price, a "YOU SAVE ₹X (Y% OFF)" line, and then 3 numbered how-to-buy steps crammed into the same bubble (e.g. "1. Buy ₹5,000 [Brand] voucher via UPI → 5% off. 2. Add item to [Merchant] cart. 3. Apply voucher → pay remaining ₹X"). Sometimes a 4th "have this cashback card?" paragraph is appended to the same bubble.
- Bubble 2 (interactive button message): a short instruction line ("Buy ₹5,000 [Brand] voucher on Gyftr using UPI") plus a tappable "Buy Gyftr Voucher" button.
- Bubble 3 (interactive button message): "Ready to buy from [Merchant]?" plus a tappable "Open [Merchant]" button.

Before these 3 bubbles, a first-time user also gets a separate welcome/onboarding text message, and every query gets a separate "Checking prices across stores..." text message while the search runs. If the product match is ambiguous, the user must also tap through a WhatsApp list-picker message (tap "Select product" → tap the correct row) before any of the above is sent. There's also a separate "See other routes" button flow for alternative merchants, which opens yet another tap-through list.

Important technical constraint: WhatsApp's Cloud API does NOT notify the business when a user taps a URL button (only taps on native quick-reply/list buttons trigger a webhook event back to us). So we cannot detect "the user completed step 1" and can't build a flow that reactively advances based on real tap confirmation for the two purchase links — any "step-by-step" redesign has to work as a one-shot sequence of messages, not a stateful wizard.

USER PROFILE: Indian, mobile-data-conscious, WhatsApp-native (many have never used a "shopping app" that isn't WhatsApp), high purchase intent (they already want to buy this specific product — this isn't discovery/browsing), price-sensitive enough to seek out a savings tool, and often not tech-sophisticated — assume some users need very explicit, can't-get-it-wrong hand-holding through a multi-step redemption process (buy a gift voucher on one site, then separately apply it at a different retailer's checkout) that has real failure modes if done out of order or incorrectly (e.g. buying the wrong voucher denomination, or checking out before applying the voucher code).

CHANGES I'M CONSIDERING (validate, refine, or challenge these):
1. Sending the actual product photo (I already have the thumbnail URL from the price-search results) as an image message with a caption, as visual confirmation before/with the price breakdown, since right now the reply is pure text with zero imagery.
2. Splitting the current single crammed "how to buy" paragraph into separate, shorter, sequential messages — one per step ("Step 1 of 2: buy the voucher" as its own message with its own button; "Step 2 of 2: use it at checkout" as its own message with its own button) instead of one dense bubble — to make a multi-step, cross-site redemption process easier to follow correctly, given we can't build a real reactive/stateful wizard (see constraint above).
3. Replacing the standalone "Checking prices..." text bubble with WhatsApp's native typing-indicator/read-receipt signal instead of consuming an extra chat bubble.
4. Only sending the welcome/onboarding message on a user's genuine first-ever interaction, not repeatedly.
